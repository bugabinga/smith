# smith-ai Crate Design

## Overview

The `smith-ai` crate handles:
1. **Provider connections** — connecting to LLM APIs
2. **Authentication** — API keys and OAuth flows
3. **Streaming** — streaming responses from providers
4. **Model registry** — all available models from bundled JSON

## Provider Data (JSON)

### Location
- Bundled at compile time: `smith-ai/src/providers.json`
- Generated by `xtask fetch-providers` from pi.dev + catwalk

## Auth Types

### 1. API Key
```rust
pub struct ApiKeyAuth {
    pub env_var: String,  // e.g., "ANTHROPIC_API_KEY"
}
```
- Read from environment variable
- Simple, stateless

### 2. OAuth
```rust
pub struct OAuthConfig {
    pub id: String,           // e.g., "github-copilot"
    pub name: String,         // e.g., "GitHub Copilot"
    pub auth_url: String,     // authorization URL
    pub token_url: String,    // token exchange URL
    pub scope: String,        // OAuth scope
    pub client_id: String,    // OAuth client ID
}

pub struct OAuthCredentials {
    pub access_token: String,
    pub refresh_token: String,
    pub expires_at: i64,      // Unix timestamp
}
```
- Requires callback server for user authorization
- Tokens stored securely (where? config file? OS keyring?)
- Auto-refresh when expired

### 3. Organization (OpenAI specific)
```rust
pub struct OrganizationAuth {
    pub org_id: String,
}
```

## Auth Resolution

```rust
pub enum AuthMethod {
    ApiKey(String),           // API key value
    OAuth(OAuthCredentials),   // OAuth tokens
    Organization {
        api_key: String,
        org_id: String,
    },
}

pub trait AuthResolver: Send + Sync {
    fn resolve(&self, provider: &str, auth_type: &str) -> Result<AuthMethod, AuthError>;
}
```

## Provider Trait

```rust
pub trait Provider: Send + Sync {
    fn id(&self) -> &str;
    fn name(&self) -> &str;
    fn base_url(&self) -> &str;
    
    fn complete(&self, request: ProviderRequest) -> ProviderStream;
    fn authenticate(&self, auth: &AuthMethod) -> Result<(), AuthError>;
}
```

## OAuth Flow

### 1. User initiates login
```rust
pub struct OAuthCallbacks {
    pub on_auth_url: Box<dyn Fn(String)>,        // Display URL to user
    pub on_code: Box<dyn Fn() -> String>,        // Get code from user (or callback server)
}
```

### 2. Login process
```rust
pub trait OAuthProvider: Send + Sync {
    fn get_auth_url(&self, state: &str) -> String;
    fn exchange_code(&self, code: &str) -> Result<OAuthCredentials, AuthError>;
    fn refresh(&self, credentials: &OAuthCredentials) -> Result<OAuthCredentials, AuthError>;
}
```

### 3. Callback server
- Starts local HTTP server on random port
- Receives OAuth callback
- Extracts code from URL parameters
- Returns code to smith

## xtask: Fetch Providers

```rust
// xtask/src/fetch_providers.rs

pub fn fetch_providers() -> Result<()> {
    // 1. Fetch from pi.dev
    let pi_models = fetch_pi_models()?;
    
    // 2. Fetch from catwalk repo
    let catwalk_models = fetch_catwalk_models()?;
    
    // 3. Merge and deduplicate
    let merged = merge_models(pi_models, catwalk_models);
    
    // 4. Generate JSON
    let json = generate_provider_json(merged);
    
    // 5. Write to smith-ai/src/providers.json
    write_providers_json(&json)?;
    
    Ok(())
}
```

## Override Cascade

```
bundled providers.json
        ↓
plugin overrides (add/modify providers)
        ↓
user config (~/.smith/providers.json)
```

### OAuth Storage

OAuth tokens stored in auth JSON file:
```json
// ~/.smith/auth.json
{
  "anthropic": {
    "access_token": "...",
    "refresh_token": "...",
    "expires_at": 1234567890
  },
  "github-copilot": {...}
}
```

### Plugin Override

Plugins can add or modify providers via SDK:
```rust
// In plugin manifest or SDK call
sdk.add_provider("my-provider", ProviderData { ... });
sdk.override_model("anthropic", "claude-3-5-sonnet-20241022", ModelOverrides { ... });
```
